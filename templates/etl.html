Ejecución ETL

En esta etapa el proceso ETL aún no se ejecuta desde la interfaz web. La carga fue realizada de manera controlada para validar la consistencia del modelo.

Funcionalidad proyectada para una siguiente versión.

Arquitectura actual

  • Backend: Flask
  • Base de datos: PostgreSQL
  • Frontend: Tabler + Chart.js
  • Modelo: Staging + Data Warehouse
  • Indicadores: vistas KPI en esquema DW

Justificación técnica

El módulo ETL permite proyectar la automatización del procesamiento de datos SIS, reduciendo la dependencia de procesos manuales y facilitando la actualización periódica de los indicadores del dashboard.

La arquitectura planteada considera una zona de carga inicial en el esquema stg, seguida de un modelo analítico en el esquema dw, donde se generan las vistas necesarias para alimentar el dashboard web.

Este módulo respalda la escalabilidad futura del sistema BI, permitiendo integrar nuevos periodos, establecimientos y fuentes de información.
